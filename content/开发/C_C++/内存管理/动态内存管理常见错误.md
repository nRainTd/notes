1. `C6011` [警告 C6011 | Microsoft Learn](https://learn.microsoft.com/zh-cn/cpp/code-quality/c6011?view=msvc-170&f1url=%3FappId%3DDev17IDEF1%26l%3DZH-CN%26k%3Dk(C6011)%26rd%3Dtrue) 在使用堆内存指针是没有验证是否为空指针，即引用了潜在的 null 指针，需要在使用前验证指针；
2. `C6308` [警告 C6308 | Microsoft Learn](https://learn.microsoft.com/zh-cn/cpp/code-quality/c6308?view=msvc-170) `realloc` 可能返回空指针：将空指针赋给 `parameter-name`（后者将作为自变量传递给 `realloc`）将导致原始内存块泄漏；
3. 对堆内存指针使用 `free` 或 `delete` 释放其指向的内存后没有将指针赋为 NULL，可能导致重复释放内存；
4. 使用 malloc 等函数分配内存指定大小的单位是字节。曾误以为是单位是类型（例如误认为 `int *` 类型的指针单位是 `sizeof(int)` ）
5. 误认为 sizeof 一个堆数组指针能获得整个数组的大小，从而填错 `malloc` 大小；
