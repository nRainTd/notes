## 1 `push` 究竟做了什么？
在我的推测中，当使用 `git push <remote> <branch>` （`git push <remote> refs/heads/<branch>:refs/heads/<branch>`）命令时，
1. git 会找到待推送的分支指针指向的提交，并依次向上寻找父提交，直到找到没有父提交的那次提交。这些提交将构成一个提交链，分支指针指向提交链中最新的那次提交。
2. 将这个提交链（附带要推送的分支指针）推送到远程仓库：
	1. 远程仓库中如果有该分支：将对比两条分支链，如果本地的只是比远程的多出一些提交，则将这些提交添加到后面，移动远程分支指针到最新提交；如果本地提交链中缺少远程提交链的一些提交，则无法成功 push 。
	2. 如果没有该分支：新建一条以该分支命名的远程分支，并从远程仓库的完整提交链中与推送来的提交链完全相同的部分的最后一次提交处分出一条岔，用它们不完全相同的部分填充这条岔，并将新建的远程分支的指针指向这条岔的最新一次提交。
