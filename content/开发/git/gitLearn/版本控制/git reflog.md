1. `git reflog` 输出的实际上是 `HEAD` 的移动记录。由于 `HEAD` 实际上是指向某次提交的指针，所以这个所谓的移动记录就是将 `HEAD` 每一次操作（`commit`、 `reset`、 `checkout/switch`）所指向的 提交 ID 记录下来，按先后排好序形成的表而已。  
2. 或者说，`git reflog` 记录了**会使 `HEAD` 指向改变的操作**的**操作记录**。
3. `git reflog` 只会记录本机的操作记录，并且**无法推送到远程仓库**；同样地，也**无法从远程仓库获取其他操作记录**。这是 `git reflog` 与 `git log` 比较显著的区别。简而言之，`git reflog` 只会记录**个人的操作记录**，并且这份操作记录**无法共享**，不能用于团队协作。
