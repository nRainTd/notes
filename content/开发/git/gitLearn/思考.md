## 何时直接 clone ，何时 flok 后再 clone ？
当连接到自己的远程仓库时，或者仅仅时拷贝而不 push 时，不需要 flok ；当需要提交 pull request 时，就需要先将项目 flok 到自己仓库，再将改动 push 到自己仓库，最后提交 pull request 。

## 如何理解 `HEAD` ？
`HEAD` 是一个指针（这里说的指针不是 c 语言里的那个指针，而是一个形象化的比喻）。它如果指向了某个分支指针，那么这个分支就是“当前分支”。而分支指针又指向某一次提交，因而 `HEAD` 总是间接指向某一次提交；对于被 `HEAD` 间接指向的提交，我们称其为“当前提交”。（其实还可以把 `HEAD` 理解为当前分支指针的别名，这样一来 `HEAD` 就是直接指向了“当前提交”）因此我们可以把 `HEAD` 简单理解为指向当前提交的指针。`git log` 、`git commit` 、`git reset` 等命令都是在当前提交的基础上进行的，也就可以理解为以 `HEAD` 为执行对象的。
